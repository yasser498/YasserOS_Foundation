# Release Rules

## Versioning

نستخدم Semantic Versioning:

```text
MAJOR.MINOR.PATCH
```

## أمثلة

- `1.0.0`: Foundation.
- `1.1.0`: Knowledge Registry.
- `1.2.0`: Design Engine.
- `2.0.0`: Project Generator.

## شروط الإصدار

كل إصدار يجب أن يحتوي على:

- Release Notes.
- Changelog.
- Deliverables.
- Acceptance Criteria.
