# Model Routing Strategy

## الهدف

اختيار النموذج المناسب حسب المهمة والتكلفة والأهمية.

## التصنيف

| نوع المهمة | النموذج المقترح |
|---|---|
| تحليل نظام كامل | نموذج قوي |
| مراجعة معمارية | نموذج قوي |
| توليد CRUD عادي | نموذج متوسط أو Codex |
| تلخيص نصوص | نموذج اقتصادي |
| توليد SQL بسيط | نموذج اقتصادي |
| مراجعة هوية تصميمية | نموذج قوي أو بصري |
| مهام محلية بسيطة | نموذج محلي عند توفره |

## القاعدة

لا نستخدم النموذج الأغلى إلا عند الحاجة.
